Summary

This sample project demonstrates how to use the BIOPAC Hardware API in C# to develop a software program that acquires and displays data from a subject while the subject views a video stimulus.

Introduction

The sample project utilizes an embedded Windows Media Player for video sitmulus playback. It uses a custom plotting alogrithm to display the data acquired from the subject using the Hardware API. When the software is launched the program prompts the user to specify a video file or a DVD drive. The specified video is then loaded into an embedded Windows Media Player. At this point, the user can choose to cue the video. Finally, the data acquisition can be started by pressing "GO."

Windows Media Player

The Windows Media Player 10 SDK does not need to be installed for this project to compile and run. However, knowing how to use the Windows Media Player Control in a C# application is critical in understanding this sample project. This information can be found in the Windows Media Player 10 SDK's documentation or on the web.

Acquisition Module

The AcquisitionModule class manages handles all the software program's interaction with the Hardware API. It acquires from channel 1, 2, and 3 from the MP36R. It assumes that the data being collected are ECG from the General Purpose Electrode, temperature from the Temperature Transducer, and the subjective response from the Variable Assessment Transducer, connected to the respective channels. The module will prompt the user to locate the Channel Preset XML file. This is located in the Hardware API install directory in the PresetFiles folder.

Data Visualization

The AcquisitionModule class contains the DrawData() function. This function takes a Graphics object where data from channel 1 is plotted as a waveform and the data from channel 2 and 3 are displayed as text. The VideoPresentation class invokes this function when the NewData event is triggered by the AcquisitionModule object. Using an event to trigger drawing will lessen the number of redraws. The NewData event is currently triggered when the length of the acquisition waveform is a 50 sample increment.
