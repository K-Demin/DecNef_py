MPCommander

Project Info

  • Language: C#
  • Development Environment: Visual Studio 2019
  • Version: 1.1.0

BIOPAC Products Used

  • MP Device: MP160 UDP, MP36R USB (source must be modified to work with MP200 if desired)

Summary

This sample project demonstrates how to use the BIOPAC Hardware API in C# to develop a command line interface to the MP device.

Introduction

The command line interface allows users to control the MP device by simply typing simple commands. When configured to communicate with the MP160 or the MP36, the user can perform simple tasks such as poll for the current values of the analog and digital channels.

Implementation

The main function of the program simply creates an instance of CommandLineManager class. When the Start() function is called the CommandLineManager object takes control of input processing and execution.

Command Line Manager

The CommandLineManager class manages all supported commands. Each user input on the console gets validated. If the first token of the input is a valid command the corresponding execute function is called. For example, if the user enters mp36usb the ExecuteMP36USB() function gets invoked. The execute functions validate the rest of the data and calls the functions from the MPDevice class.

MP Device

The MPDevice class is the base class for the MP36USB and MP160UDP classes. It abstracts the differences between function calls to the Hardware API when using the MP36R or the MP160. The CommandLineManager class contain a MP variable of type MPDevice. When the user invokes the mp36usb or the mp160udp command, an instance of the corresponding class is assigned to the MP variable.

Expansion

The MPCommander program does not support the full Hardware API command set. Additional Hardware API commands can be exposed through this program by studying and modifying the source code.

Notes

  • MPDevImports.cs is the C# wrapper for the Hardware API's methods
