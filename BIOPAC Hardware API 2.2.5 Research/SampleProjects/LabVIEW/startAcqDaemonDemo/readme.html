Summary

This sample project demonstrates how to acquire data from the MP device in LabVIEW using the BIOPAC Hardware API. There are three VIs in this project. The VI startAcqDaemonDemo160 is configured for use with the MP160, VI startAcqDaemonDemo150 is configured for use with the MP150, and the VI startAcqDaemonDemo36 is configured for use with the MP36R.

Figure 1: Configuration Section of the Front Panel

Figure 2: Block Diagram key portion

Introduction

This project utilizes the BIOPAC Hardware API functions startAcqDaemonDemo() and receiveMPData() to acquire data in real time. With these two functions, it is possible to acquire, analyze, and plot data in real time without losing data points.

Front Panel

The Front Panel is comprised of three Waveform Charts that are hard wired to plot data from 3 analog channels. It also has a configuration section that must be modified for your MP device configuration (see Figure 1). After running the VI, configure the DLL, configure the acquisition, and then press the "Start" button. This will initiate the acquisition. The VI will continue plotting to the Waveform Charts until the user toggles the "Start" button to the off position or an error occurs.

Block Diagram

The Block Diagram consists of several Call Library Function Nodes. These nodes interface with the BIOPAC Hardware API. Figure 2 is the key section of the block diagram. It starts the acquisition daemon, starts data acquisition, calls receiveMPData() function and plots the samples in the appropriate Waveform Charts in real time.

Notes

  • There are several ways of calling Shared Libraries (DLLs) from LabVIEW. This project uses the Call Library Function to call functions from the BIOPAC Hardware API in LabVIEW. Please read LabVIEW's documentation regarding usage of the Call Library Function.
