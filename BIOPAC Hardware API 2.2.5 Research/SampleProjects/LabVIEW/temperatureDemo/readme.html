Summary

This sample project demonstrates how to use the BIOPAC Hardware API in LabVIEW to develop a temperature display system. This system utilizes BIOPAC's temperature amplifier and transducer to acquire the current temperature. It can be configured to acquire data from the MP36R, MP160, or MP150.

Figure 1: Block Diagram key portion

Introduction

This project utilizes the BIOPAC Hardware API function getMostRecentSample() to poll for the current temperature. If the current temperature exceeds a threshold, the specified voltage level is sent to the specified Analog Output.

Front Panel

The Front Panel is comprised of a Thermometer, a Dial and a Pointer Slide. The Thermometer displays the current temperature. The Dial sets the temperature threshold. The Pointer Slide sets the voltage level to output. After running the VI, configure the DLL, configure the acquisition, and then press the "Start" button to initiate the acquisition.

Block Diagram

The Block Diagram consists of several Call Library Function Nodes. These nodes interface with the BIOPAC Hardware API. Figure 1 is the key section of the block diagram. It starts the data acquisition, calls the getMostRecentSample() function, and controls the main logic of the VI. When configured to use the MP160 or MP150, the Temperature Amplifier (SKT100C) is assumed to be set to Analog Channel 1. When configured to use the MP36R, the Temperature Transducer (SS6L) is assumed to be attached to Analog Channel 1. The voltage to Fahrenheit conversion is done inside the Hardware API. It is configured with a call to setAnalogChScale() function.

Notes

  • There are several ways of calling Shared Libraries (DLLs) from LabVIEW. This project uses the Call Library Function to call functions from the BIOPAC Hardware API in LabVIEW. Please read LabVIEW's documentation regarding usage of the Call Library Function.
