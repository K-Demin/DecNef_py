Summary

This sample project demonstrates how to create a list of all MP160s/MP150s in the network using the BIOPAC Hardware API.

Figure 1: Block Diagram key portion

Introduction

These two projects utilize, in one case, the BIOPAC Hardware API functions findAllMP160() and readAvailableMP160SN() to create a list of MP160s in the network, and in the other case, the corresponding MP150 functions to create a list of MP150s in the network.

Front Panel

The Front Panel is comprised of a Text Indicator and a Push button. After running the VI, press the "Start" button and all the serial numbers for available MP160s/MP150s in the network will appear in the Text Indicator.

Block Diagram

The Block Diagram consists of several Call Library Function Nodes. These nodes interface with the BIOPAC Hardware API. Figure 1 is the key section of the block diagram. The while loop calls on readAvailableMP160SN() or readAvailableMP150SN() to read an array of characters and append it to the Text Indicator.

This VI passes an invalid MP160/MP150 serial number to the function connectMPDev(). This is a specification requirement, in order to call the function findAllMP160() or findAllMP150() successfully.

Notes

  • There are several ways of calling Shared Libraries (DLLs) from LabVIEW. This project uses the Call Library Function to call functions from the BIOPAC Hardware API in LabVIEW. Please read LabVIEW's documentation regarding usage of the Call Library Function.
