Summary

This sample project demonstrates how to acquire data from the MP device in LabVIEW using the BIOPAC Hardware API. There are three VIs in this project. The VI getBufferDemo160 is configured for use with the MP160, the VI getBufferDemo150 is configured for use with the MP150, and the VI getBufferDemo36 is configured for use with the MP36R.

Figure 2: Block Diagram key portion

Introduction

This project utilizes the BIOPAC Hardware API function getMPBuffer(). According to its specification, this function does not allow for real-time data processing, analysis or plotting. However, it is good for acquiring data for a short interval and at a high sampling rate.

Front Panel

The Front Panel is comprised of three Waveform Charts that are hard wired to plot data from 3 analog channels. It also has a configuration section that must be modified for your MP device configuration (see Figure 1). In accordance to the specification of the getMPBuffer() function, the number of samples to acquire must be specified. After running the VI, configure the DLL, configure the acquisition, and then press the "Start" button to initiate the acquisition. The VI will block until the desired number of samples has been acquired or an error occurs.

Block Diagram

The Block Diagram consists of several Call Library Function Nodes. These nodes interface with the BIOPAC Hardware API. Figure 2 is the key section of the block diagram. It starts the data acquisition, calls to the getMPBuffer() function and plots the samples in the appropriate Waveform Charts. Since there are three active channels, the VI allocates an array that can hold three times the number of samples. The array that the getMPBuffer() function returns is interleaved and must be decimated.

Notes

  • There are several ways of calling Shared Libraries (DLLs) from LabVIEW. This project uses the Call Library Function to call functions from the BIOPAC Hardware API in LabVIEW. Please read LabVIEW's documentation regarding usage of the Call Library Function.
