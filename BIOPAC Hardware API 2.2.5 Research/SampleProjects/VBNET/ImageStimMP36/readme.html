ImageStim

Project Info

  • Language: Visual Basic .NET
  • Development Environment: Microsoft Visual Studio .NET 2019
  • Version: 1.1.0

BIOPAC Products Used

  • MP Device: MP36R USB
  • Electrode: General Purpose Electrodes (EL503)
  • Electrode Lead: Shielded Lead Set (SS2L)
  • Transducer: Eight-channel Marker Box, Digital (SS55L)

Summary

This sample project demonstrates how to use the BIOPAC Hardware API in VB .NET to create an image rating system that also records the subject's physiological and subjective repsonse.

Figure 1: XML output file viewed in a web browser with XML Stylesheets support.

Introduction

The sample project displays a set of images to the subject while it acquires and records data. Each image is displayed for approximately five seconds. While the image is displayed, the subject must rate the stimulus using the Eight-channel Marker Box. The software also acquires and records data from Analog Channel 1 of the MP36R.

Usage

When the program is launched, the user must specify a folder where images are stored. The software only accepts images with file extension, jpg, jpeg, and gif. The user must also specify the directory where it will save the output files. When the "GO!!!" button is pressed, the user will be prompted to locate the Channel Preset XML file. This is located in the Hardware API install directory in the PresetFiles folder. From there, the images are displayed and data acquisition begins.

Program Logic

If all the configuration routines are successfully completed, the software follows this logic for each image:

  • Display the image
  • Start the acquisition
  • Get the first 5 seconds of data
  • Clear the screen
  • Stop the acquisition
  • Extract Analog Channel 1 from the data buffer
  • Extract the subjective score from the data buffer

Data Acquisition

The software uses getMPBuffer() function from the Hardware API to acquire data. Since getMPBuffer() function is a blocking function, it is not necessary to time the number of seconds the image has been displayed. The software simply requests 5 seconds of data from the getMPBuffer() function immediately after the image is displayed and the start of acquisition.

Data Reporting

All the data from each stimulus is saved in memory until all the images have been displayed. A report is generated from the serialization of the StimulusReport class into an XML file. The waveform from Analog Channel 1 is saved as an image into the specified directory. The software also saves a copy of XML Stylesheet that was compiled as a resource in the executable. When the output XML file is viewed in a web browser that supports XML Stylesheets, it will be displayed similar to Figure 1. The XML Stylesheet directs the browser to display the data from the XML file in a table and sort it from highest to lowest subjective rating.
