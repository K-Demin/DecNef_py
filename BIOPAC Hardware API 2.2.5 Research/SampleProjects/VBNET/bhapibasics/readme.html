Summary

This sample project demonstrates the basic usage patterns the BIOPAC Hardware API in VB .NET.

Introduction

The sample project is a console application that demonstrates how to use the getMPBuffer() and getMostRecentSample(). It also shows how to create an application that supports the MP36R and the MP160 using the Hardware API.

Usage

When the program is launched, the user is prompted to enter "MP200UDP", "MP160UDP", "MP36USB" or "EXIT." The program then configures itself to run the demo for the MP200, the MP160, the MP36R or exit the application. The program will execute until it acquires data using poll for the most recent sample ten times. It then starts a new acquisition and gets the first two seconds of data. Finally, it outputs some of the values acquired on the console.

MP36R and MP160

The Hardware API abstracts most of the differences between the MP36R and the MP160. However, the Hardware API users must account for some of the physical differences between the two MP devices if they want to support both devices like the number of analog and digital channels. This sample project shows how to programatically handle the differences in the number analog and digital channels of the devices.

Notes
