BIOPAC Hardware API 2.2.5 for Windows Sample Programs

  1. The sample application files are located at 'C:\Program Files\BIOPAC Systems, Inc\BIOPAC Hardware API 2.2.5 Research\SampleProjects'
  2. MPDEV libraries have been compiled with both Microsoft Visual Studio (MSVS) 2010 and MSVS 2019. The compiled libraries generated by the two compilers may be found in 'C:\Program Files\BIOPAC Systems, Inc\BIOPAC Hardware API 2.2.5 Research\VC10' and 'C:\Program Files\BIOPAC Systems, Inc\BIOPAC Hardware API 2.2.5 Research\VC14', for MSVS 2010 and 2019 respectively. Libraries have been compiled for both 32-bit and 64-bit systems by both compilers, so there are four sets of library files. Use the appropriate set for sample applications and for your own programs.
  3. Sample applications in C++, C# and VBNet are provided as 64-bit executable (.EXE) files made with MSVS 2019 (VC14).
    In order to run the sample applications:
    1. Copy mpdev.dll and xerces-c_3_1.dll from 'C:\Program Files\BIOPAC Systems, Inc\BIOPAC Hardware API 2.2.5 Research\VC14\x64' into a folder that includes a sample application executable file (*.exe).
    2. Double click on the sample application executable file
    3. Some sample applications will work with either the MP36R, MP36A, MP200, MP160, or MP150, but some are hardware specific.
      • Applications specific for MP160 can generally be made to function for MP150 or MP200 by changing a "160" to a "150"/"200" in a small number of code lines and then re-building the project to create a new executable file.
      • Applications specific for MP35 can generally be made to function for MP36 or MP36A by changing a "35" to a "36"/"36A" in a small number of code lines and then re-building the project to create a new executable file.
  4. Sample applications for LabView, MATLAB, and Python must be run from within those programs. Follow the links below for each sample project and see comments in sample project source code for more details about each specific sample.
  5. MP devices should be connected and powered on prior to starting sample applications.

C/C++

mp1XXdemo: Demonstrates basic usage of the BIOPAC Hardware API's methods.

C#

Biofeedback: A simple biofeedback application that uses BIOPAC Hardware API.

GoalKick: A simple game that uses the BIOPAC Hardware API and a transducer as a human interface.

MPCommader: A command line interface to the MP160 and MP36R using the BIOPAC Hardware API.

TemperatureControl: A temperature control system that uses the BIOPAC Hardware API, BIOPAC transducer and common electronic parts.

VideoStimulus with MP36R device: A video stimulation application that uses the BIOPAC Hardware API to acquire data from MP36R device.

LabVIEW

getBufferDemo: Demonstrates how to acquire data from MP160, MP150, or MP36R using BIOPAC Hardware API's getMPBuffer() function.

listAllMP160Demo\listAllMP150Demo: A pair of VIs that list all MP160s/MP150s in the network

startAcqDaemonDemo: Demonstrates how to acquire data from MP160, MP150, or MP36R using BIOPAC Hardware API's startAcqDaemonDemo() and receiveMPData() functions.

temperatureDemo: A VI that monitors the temperature and outputs voltage to a specified Analog Output given a threshold.

MATLAB

mpdevdemo: Demonstrates basic usage of the BIOPAC Hardware API's methods.

Python

spacegrip: Demonstrates basic acquisition of data using BIOPAC Hardware API's methods.

VB .NET

bhapibasics: Demonstrates the basic usage patterns of the BIOPAC Hardware API.

FunctionGenerator: A program that transforms an MP device to an arbitrary waveform generator using the BIOPAC Hardware API.

ImageStim with MP36R device: An image stimulation program that acquires data from MP36R device using the BIOPAC Hardware API.
